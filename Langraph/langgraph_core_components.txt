Common LLM Worklows:

* Prompt Chaining --> calling LLMs multiple times.
* Routing         --> LLM Call Router (Decision maker) Routes the request to the respective LLM across multiple LLMs.

* Parallelization: Given task breaks down into multipke sub-tasks, executes them simalteneously to get their respective outputs to aggregate (Aggregator) them as one final output.

* Orchestrator Workers:
Task  --> Multiple || sub-tasks based on the query, decision making is done by orchestrator which task to do by which LLM to get the results from each sub-task to aggregate the final output.


* Evaluator Optimizer:

Graphs, Nodes & Edges: 
Node: It is a python function. It represents each task. What to execute.
Edges: It connects the nodes. It decides the flow. When to execute.

State: It is the shared memory that flows through the LLM workflow. It holds all the data that passed through nodes as the graph runs.
It is stored in key-value pairs.
It is accessible to all the nodes in the graoh.
State is mutable.


Reducers: It defines how updates from nodes are applied to the shared state.
Each key in the state can have its own reducer, which determines whether new data replaces, merges or adds to the existing value.
